## Comité Técnico

El comité técnico Express se reúne en línea cada dos semanas (según sea necesario) para discutir el desarrollo y el mantenimiento de Express,
y otros problemas relevantes para el proyecto Express. Cada reunión se anuncia típicamente en un asunto
[expressjs/debates](https://github.com/expressjs/discussions/issues) con un enlace para unirse o ver la reunión, que está
abierto a todos los observadores.

Las reuniones están registradas; para una lista de las grabaciones, vea el [canal de YouTube Express.js](https://www.youtube.com/channel/UCYjxjAeH6TRik9Iwy5nXw7g).

## Express está hecho de muchos módulos

Nuestra vibrante comunidad ha creado una gran variedad de extensiones,
[módulos de middleware](/en/resources/middleware) y marcos de alto nivel.

Además, la comunidad Express mantiene módulos en estos dos órganos de GitHub:

- [jshttp](https://github.com/jshttp) módulos que proporcionan funciones útiles de utilidad; vea [módulos de utilidad](/en/resources/utils).
- [pillarjs](https://github.com/pillarjs): low-level modules that Express uses internally.

Para mantenerte al día con lo que está pasando en toda la comunidad, revisa el [ExpressJS StatusBoard](https://expressjs.github.io/statusboard/).

## Problemas

Si te encuentras con lo que piensas que es un error, o simplemente quiere hacer
una solicitud de característica abrir un ticket en la [cola de incidencias](https://github.com/expressjs/express/issues).

## Ejemplos

Ver docenas de aplicaciones Express [examples](https://github.com/expressjs/express/tree/master/examples)
en el repositorio que cubre todo, desde diseño API y autenticación hasta integración de motores de plantillas.

## GitHub Discussions

La sección [Discusiones de GitHub](https://github.com/expressjs/discussions) es un excelente espacio para participar en conversaciones sobre el desarrollo y mantenimiento de Express, así como compartir ideas y discutir temas relacionados con su uso.
